# Git Branch
1. master 브랜치는 항상 Deploy 할 수 있는 상태로 두자.
2. 작업할 때는 항상 Topic 브랜치를 만들어서 작업한다.(기능 구현)
3. 브랜치 이름은 누가봐도 알 수 있도록 자세히 짓자
4. 정기적으로 자주 Push 하자 ( 로컬 컴퓨터가 언제 날라갈지도 모른다)
5. Merge가 끝나면 해당 Topic 브랜치는 바로 삭제하자.
